| Version | Supported          |

## Reporting a Vulnerability

If you discover a security vulnerability in this repository (e.g., sensitive data exposure, misconfigured access controls, or PII in datasets), please report it responsibly.

**Do NOT open a public GitHub Issue for security vulnerabilities.**

### How to Report

1. Email: **jmsernatuya@gmail.com**
2. Subject: `[SECURITY] <repository-name> - Brief description`
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if available)

### Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 7 days
- **Fix or Mitigation**: Within 30 days for critical issues

### Scope

This policy covers:
- Data files containing unintended personally identifiable information (PII)
- Broken or insecure links in documentation
- Incorrect licensing or attribution that could violate third-party rights

### Out of Scope

- Issues with GitHub's own platform security
- Theoretical vulnerabilities without proof of concept
